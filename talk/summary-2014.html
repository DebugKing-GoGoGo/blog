2014年终总结

从 2012 年开始，我便开始每年都写年终总结，去年东施效颦某些前端大牛，将一篇洋洋洒洒的总结写在了 Github 的 issue 里，随着某次情绪不稳定，删掉了该 repo 之后，才发现 issue 并不提供 git 机制，悔恨不已。 从而莫名其妙的丢掉了一年的总结，每每想到这件事，就感觉仿佛 2013 年未曾活过一般。

但实际上，2013 年也的确没有 2012 年那么曲折，也没有 2014 年这般炼狱。

何谓炼狱？说来话长，今年大体可分为两个阶段:

  1. 继续给采集擦屁股
  2. 进入三维图形领域

第一阶段，恰为上半年，约莫到4月中旬；第二阶段从5月开始，直到年末。

如果说去年对于数据采集的总结，达到了我个人对于这部分工作理解之高峰的话，后续的工作只能称得上是一路滑向山谷。这也是当一个软件逐步做大，参与的人员逐步变多，所带来的必然结果。 第一版与第二版，几乎是我一个人把控，只需要做好 KISS 的原则即可。而到了第三版，领导决定对其增加许多额外的功能与职责，并举全组之力，希望其能够称之为一个完整的解决方案，坚持 KISS 就显得有些不切实际了。 只记得在 2013 年 10 月到 12 月份，无尽的讨论并未产生良好的设计，而是诞生了一个不断妥协的结果：与上层接口的妥协，与调用方的妥协，与领导的妥协，与行业经典软件的妥协。可以想象这样一个怪东西会有怎样的结局了。

所以到了 2014 年，我过足了"设计的瘾"，开始回归到具体实现上来了。说说接触到的一些新的尝试：

  1. 在 MFC 中，界面美化一直是一个繁复的过程，我时常倾羡一些成熟软件的外观，直到我看到了这篇文章。从此发现 Qt 的威力，模仿并草创了采集的主体界面。
  2. 如何实现 C/S 程序与 B/S 程序的交互？(也可以理解为 C++ 与 Java Web 的交互)，我接触到了gSOAP，明白了如何用 C++ 来写 Web Service.
  3. 几个小玩意：C/S 集成页面，即集成多个 exe 到一个应用中；单位换算机制，即公制-英制-自定义单位的快速转换。
  4. 丰富 DAS 采集框架，主要完善消息总线的设计，尽可能的简化结构，去除不必要的内部消耗。这里面我温习了不少设计模式的知识，深入了解了观察者模式与工厂系的模式。

其余的工作便是改 Bug，完善底层架构，改 Bug，完善。。。

这个阶段应该算是对三年采集工作的一次梳理和总结，另外也尝了 Qt 的鲜，真正做到了彻底摆脱 MFC，同时了解了一些关于 Web Service 协议的东西。

这个阶段，我应该算是全力扑在了工作上，但由于工作中需要将以往的仪器的适配 DLL 全部从 C++ 改为纯 C 语言，所以业余时间我也开始重读 C 程序设计语言，第一次开始认真做习题。

第二阶段，实际从 3 月份即开始蠢蠢欲动了，但直到 4 月中旬这事才板上钉钉，那次 工作变动 对我真的影响很大。我开始比较正式的学习 Qt 与 OSG，从 2D 绘图开始，到 3D 中画出第一个球体。 这部分经历记忆犹新，不同于以前应用软件设计的领域，图形学对我来说，真的是完全陌生。其中的压力与艰辛，实在只能意会不可言传。也正是从那时起，我开始混迹知乎，读 C++ GUI Qt4编程。

期间慢慢适应了新的工作环境，但也遭遇了史上最低的一次绩效考核，情绪波动很大，很长时间里非常自暴自弃，认定自己是个 loser，现在想想有些可笑，但负面的情绪不仅影响了工作热情，也影响了生活，甚至影响了与女友的关系。

而一次非常意外的脑子犯二经历，让我重新捡起了 C++ Primer 这本神奇的书。 又让我知道了王月的那个习题答案的 repo，于是约莫从八月开始，我便开始认真阅读该书，并尽量完整的补充习题答案。

最初，这件事对于我的意义可能只是烦闷工作的一个消遣，也同时能够满足我长期以来跃跃欲试的"编纂情节"，我曾为其设计了漂亮的 web 页面，包括设置了评论框，连字体、CSS 样式，都是花过心思的。 但后来随着我越来越深入的了解 C++ 11 的新规范，才发现了一个全新的世界，并清晰的看到自己基础的薄弱。以至于后来我撤掉了华丽的 web 页面，踏踏实实的研究 C++ 本身，并从始至终的贯彻书中作者无意间提到的一句名言：

Brevity Can Be a Virtue

也是从那时起，我开始打理我的豆瓣主页, 粗略的模仿王月的读书进度条(现在已经撤掉了)，并开始写 C++ Primer 的读书笔记。 正如我在某个答案的最后所说，我始终相信，能够啃下大砖头的人，必然是人中龙凤，而我当时竟没曾预料到这么快就遇上了一个。 这对于我有很大的触动，于是，很庆幸，我没有再一次中途放弃，而是坚持将这个计划继续了下去。虽然进展很缓慢，截止到今天，整整 4 个月，我才进展到 12 章，其中有过烦闷，欢心，自怨，骄傲，等等心绪。

有了第一次义无反顾的坚持，第二次便来的很容易，从 10 月 15 日开始，我开始每天一道 LeetCode，起初这也只是一种好奇驱动而已。 但随着题量的提升，涉及到数据结构、算法等等一系列知识时，我才真正的明白面试题的意义（以往我是对此深恶痛绝的）。做这个的时候我也是无意间列了一个时间表， 而截止到今天，我竟然依然按照该表在执行。

上述两件事，对于其他人并没有什么太大的意义，但对于我来说，却是非常大的改变。我从来就不是一个善于坚持的人，有生以来，我唯一坚持做下来的事情，也许就是上述两件了。 这对于 C++ 、数据结构、算法的学习可能微不足道，而对于个人修身的影响却是巨大的。MacTalk 的池大大曾表示，每天坚持写 MacTalk ，看似是一件小事，却非常的不易。我也是至今才能够深刻体会。

更不用说这两件事带给我其他的好处了，捡重点说两个吧：

  1. 我已经离不开 Github 了，并在上面认识了很多厉害的朋友。
  2. 我竟然开始学着使用 StackOverflow 了，长期以来我都是该站的得益者，今年我成为了一个贡献者。

另外的一些变化还在于：

  • 我越来越不喜欢夸夸其谈，喜欢用代码说话，可以看到知乎上我早期的答题都是谈经验，后来都是直接拼代码，后者是需要勇气的，因为是毫不遮掩的接受大家的批评。
  • 我从忽略C++基础语法问题，到后来非常喜欢回答C++基础语法问题，再到现在不愿花功夫回答基础语法问题。(倒是有点看山不是山的感觉)
  • 我从喜欢刷微博刷知乎，变的喜欢刷 StackOverflow 和 segmentfault，当然后者并未发展起来，但我已经贡献了许多答案了。

相比之下，这个阶段我的工作倒是显得不怎么出彩了，期间重点研究的 LOD 与 Octree 问题到现在也没有太多的头绪，倒是因为解决了几个其他的三维显示问题，从而拿到了公司内部的一个小奖(技术达人)。 其奖金还算丰厚，也弥补了一点绩效不好看的遗憾。

最后一部分总结，要留给一直陪伴我的女友。今年我们一起走过了相识的第二个年头。上面提到的那本 C++ Primer 5th 的英文原版，就是她从英国给我带回来的，很惭愧，那时国内该书还未曾有中文版， 而她得知我是搞 C++ 的，便偷偷给我买了此书，那时我便发誓要将此书读个通透，没想到今年才开始行动。也是她，帮助我克服了对于大部头英语的恐惧，对于英语书写的恐惧，甚至是口语上的恐惧。

我想，英语对于一个程序员的意义，是不言而喻的。同理，她对于我的意义，也是无法估量的。

生活里的我向来笨手笨脚，这一点给她带来了不少烦恼。她是一个在生活上很会花心思的人，而我则基本是个执着的工作狂，如此直接的矛盾，给我俩带来了不少争吵，最终都以我死皮赖脸的道歉结束。 后来我们曾沟通多次，我才逐渐开始意识到自己这样"忙碌"的生活本质上是一种无序的混乱以及思维的懒惰。我开始关注极简主义，开始逐步砍掉一些无意义的事情。这对于我来说，算是一个不小的考验，毕竟积习难改。 但我已将之作为我明年的目标。规律化的生活，分配好各项事宜的时间，应该算是一项成熟的标志。

她对我的影响，往往是润物细无声的，从穿着打扮，到饮食习惯，再到对品质的追求。而最大的影响，是让我看到了更大的世界，听到了更多不同的声音，从以往的一意孤行，到现在的包容并蓄，这都是她给我带来的宝贵财富。 我的蠢笨与她的聪颖，以及完全截然不同的两种性格，竟然走到一起两年之久，也不得不叹服缘分的力量了。

最后，虽然有很多现实问题摆在我俩面前，但我还是希望今年能够缘上加缘，携手组建一个温馨的家庭。

今年的吃过的亏很多，教训也很多，主要体现在以下几条：

  1. 注意时间陷阱，要善于给自己设置时间节点，切记不要无休无止。
  2. 注意判断取舍，什么该做，什么不该做，有自己的标准。这一点直接影响上面的时间陷阱。
  3. 注意读书方法，第一遍尽量利用(kindle + 闲散时间)，标记重点；第二遍针对重点温习；然后实践中验证。
  4. 注意抓住本质，学习不是为了克服遗忘，而是为了遗忘。遗忘形式上的东西，把握核心本质。
  5. 注意归纳总结，今年的总结还是不够，总结代表思考，不经总结的学习，像是用漏斗乘沙。
  6. 注意还是坚持，坚持带来的最大的好处，是心态的稳定，不以物喜，不以己悲，按部就班做好该做的。

这是今年仍有欠缺的地方，明年继续努力。

  • 多一个坚持：每周一篇技术博客。
  • 力求在年底前，看完三本书：CSAPP + SICP + CLRS
  • 学历提升，与换工作，二选一。
